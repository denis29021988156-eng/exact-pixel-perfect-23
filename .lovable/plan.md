## 1. Сделать AI-сводку «продающей» и практичной

**Проблема:** AI выдаёт «всё плохо», шаблонные лозунги, неверная интерпретация Risk Index 10/100 как «максимальной угрозы».

**Правки в `supabase/functions/city-briefing/index.ts` и `city-copilot/index.ts`:**
- Переписать system prompt в управленческом тоне:
  1. Сначала **«Что под контролем»** (закрытые инциденты, стабильные зоны, % выполнения по проектам).
  2. Затем **2–3 точки внимания**, не больше.
  3. В конце **конкретные шаги** в формате «Поручить [ФИО / департамент] — [действие] — срок [дата]».
- Чёткая шкала Risk Index в промте: 0–30 низкий («штатный режим»), 30–60 умеренный, 60–80 повышенный, 80+ критический. Запретить слова «чрезвычайная ситуация», «кризисное управление», «мобилизация резерва».
- Использовать имена ответственных и адреса из контекста, а не общие отрасли.

## 4. AI знает сотрудников и быстро формирует поручение

**4.1. Заполнить тестовых сотрудников.** Миграция `UPDATE` для существующих профилей: проставить department, position и осмысленные ФИО:
- `Сергей Иванов` → нач. отдела ЖКХ, department=`utilities`
- `Петр Иванов` (1-й) → нач. отдела транспорта, `transport`
- `Петр Иванов` (2-й) → нач. отдела благоустройства, `improvement`
- `Иванов Иван Иванович` → нач. отдела соц. сферы, `social`
- `Тестовый Мэр` → нач. отдела строительства, `construction`
- `Денис` (admin) — не трогаем
- `Мэр Города` — не трогаем

После этого по одному ответственному на каждый из 5 департаментов.

**4.2. Контекст в copilot (`city-copilot/index.ts`):** подгружать `profiles + user_roles` (только role IN ('employee','deputy')), вставлять в system prompt блок «СОТРУДНИКИ МЭРИИ» с user_id, ФИО, должностью, департаментом — чтобы AI рекомендовал конкретного исполнителя.

**4.3. Структурированное предложение поручения.** Расширить уже существующий протокол MEMORY_SAVE: AI отдаёт строку
```
TASK_SUGGEST: assignee_user_id=<uuid>; title=<...>; deadline=<YYYY-MM-DD>; department=<...>
```
Фронт парсит её и **скрывает** из видимого ответа (как сейчас MEMORY_SAVE).

**4.4. UI быстрого подтверждения в `src/components/CityCopilot.tsx`:**
- Под сообщением AI с распарсенным `TASK_SUGGEST` рендерим компактную карточку: исполнитель, текст, срок, и две кнопки **«Отправить»** / **«Изменить»**.
- «Отправить» → `supabase.from('tasks').insert({ assigned_to, title, deadline, department, responsible: <ФИО>, created_by_name: <текущий user> })`. Toast «Поручение отправлено», в чат добавляется системное сообщение «✓ Поручение зафиксировано».
- «Изменить» → открывает существующий `CreateTaskDialog` с предзаполненными полями.

**4.5. Quick-команда** в `quickCommands` копилота: «Дать поручение» → отправляет «Предложи поручение по самому критичному инциденту: кому, что, к какому сроку». AI отвечает с `TASK_SUGGEST`, мэр жмёт «Отправить».

**Безопасность:** insert в `tasks` подчиняется RLS (`Tasks insert mayor or deputy of dept`). Для мэра это работает; для deputy — только в свой департамент. Если AI предложит чужой департамент — RLS отклонит, покажем понятный toast.
