## Шаги реализации

### 1. БД: добавить роль `admin` (миграция)
- Расширить enum `app_role` значением `'admin'`.
- Обновить RLS-политики:
  - `user_roles` — управление пользователями: `admin` (вместо `mayor`).
  - `sla_matrix_draft` — manage только `admin`.
  - `audit_log` — read для `admin` (вместо `mayor`).
  - `feature_flags` — manage для `admin` (вместо `mayor`).
  - `data_sources`, `staging_raw`, `ingestion_log`, `telegram_messages` — добавить `admin` к существующим правам.
- На «городские» таблицы (incidents, projects, contracts, escalations, weather_alerts, benchmarks, media_mentions, public_complaints) — `admin` НЕ добавляем.
